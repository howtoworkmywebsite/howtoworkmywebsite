üîó How Website Navigation Works

Navigation is how users move between different pages on a website. There are several types:

üìÑ Multi-Page Navigation

Traditional method: Each link loads a completely new HTML file from the server.

Example:

home.html ‚Üí about.html ‚Üí contact.html

‚ö° Single-Page Application (SPA)

Modern method: One HTML file, JavaScript changes content dynamically without reloading.

This website uses SPA!

üîó Anchor Links

Same page navigation: Jump to different sections on the same page.

Example:

#top ‚Üí #middle ‚Üí #bottom

üåê Understanding URLs

URL stands for Uniform Resource Locator - it's the complete address of a webpage.

URL Structure Breakdown:

https://www.example.com:443/blog/article?id=123#section2
https://

Protocol - How to communicate with the server

‚Ä¢ http:// - Not secure
‚Ä¢ https:// - Secure (encrypted)

www.example.com

Domain Name - The address of the website

‚Ä¢ www - subdomain (optional)
‚Ä¢ example - domain name
‚Ä¢ .com - top-level domain (TLD)

:443

Port Number - Which door to use (usually hidden)

‚Ä¢ Port 80 for HTTP
‚Ä¢ Port 443 for HTTPS

/blog/article

Path - Which page or file to load

Like folders on a computer: /folder/subfolder/file

?id=123

Query Parameters - Extra information sent to the page

Used for searches, filters, tracking, etc.

#section2

Fragment/Anchor - Jump to specific part of the page

Scrolls to an element with id="section2"

‚òÅÔ∏è How Web Hosting Works

Web hosting is like renting space on a computer that's always connected to the internet. This computer is called a server.

üíª

Your Browser

(Client)

Sends request:
"Give me example.com"

‚Üí
üåê

Web Server

(Host)

Sends back:
HTML, CSS, JS files

üîÑ The Complete Process:

  1. You type a URL in your browser (e.g., www.example.com)
  2. DNS Lookup: Your browser asks "Where is example.com?" and gets an IP address (like 192.168.1.1)
  3. Connection: Your browser connects to that server computer
  4. Request: Your browser says "Send me the homepage please"
  5. Response: The server sends back HTML, CSS, and JavaScript files
  6. Rendering: Your browser displays the website
